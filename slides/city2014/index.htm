Five essentials:

An interactive data visualisation must:

  • Tell a story...
  • ...that cannot be told using simpler means
  • Use data analysis integral to the reporting ...
  • Be of lasting value
  • Involve visual journalists from the start.

1. Tell a story

An interactive news application must meet a clearly-stated user need and explain a topic with a narrative structure, not just display data for the reader to explore.

2. Eliminate simpler solutions

Reserve interactivity for essential cases.

Text and static graphics are:

  • faster and easier to produce than custom code
  • more universally accessible across platforms and devices

Avoid ‘advent calendars’.

3. Use data integral to the story

The data being displyed was analysed to find the story.

It must not consist primarily of data selected after the fact to illustrate a conclusion reached by other reporting techniques.

4. Build something of lasting value

Introduce functionality or tell an ongoing story that can attract readers for longer than any one news item, ideally indefinitely.

Rule of thumb: It should retain value to readers for longer than it takes to produce.

5. Involve specialists from the start

The ‘deli counter’ model no longer works.

... and at least one of these:

  • Provides a narrative with both a ‘macro’ and a ‘micro’ view
  • ...ideally ‘personal relevancy’
  • Automates an repetitive editorial process
  • Integrates social media
  • Uses data outputs beyond graphics
