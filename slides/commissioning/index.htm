What data do we need?

What is the unit of analysis?

  • Stores

What must we know about each store?

  • Type of store
  • Location
    • Compete address
    • Lat-Lon coordinates
  • Date of opening
  • Name or unique identifier

Get granular!

Don’t let your sources aggregate data for you

  • Print space constraints don’t apply.
  • Go beyond the ‘top-line’ numbers.
  • Always seek the most granular data possible.

Re-phrase questions for granular data

  • “How many stores are there?”
  • “How many stores opened in each year since 1970?”
  • “How many stores opened in each state each year since 1970?”
  • “At which precise location and precise date did each store open?”
  • “Just gimme the whole damn spreadsheet; I can work it out myself!”
  • “Do you have that as a CSV?”

Who has granular data?

  • Companies
  • Governments
  • Analysts
  • Academic researchers

Recognising raw data

Look for files like this:

  • XLS: Excel Files
  • CSV: Comma-seperated values
  • TSV: Tab-seperated values
  • SQL: Structured Query Language
  • SPSS, SAS, Stata, R
  • XML: eXtendable Markup Language
  • JSON: Javascript Object Notation

Maps are data too!

Maps are data visualisations, not something you ‘draw’. You need data to make them, so ask for:

  • SHP: Esri Shapefile
  • KML: Keyhole Markup Language
  • GeoJSON: Geographic JSON
  • CSV: CSV of lat/lon coordinates

Ask press officers to forward requests to mapping departments

Five characteristics we insist upon:

An interactive or news application must:

  • Tell a story...
  • ...that cannot be told using simpler means
  • Use data analysis integral to the reporting ...
  • Be of lasting value
  • Involve visual journalists from the start.

1. Tell a story

An interactive news application must meet a clearly-stated user need and explain a topic with a narrative structure, not just display data for the reader to explore.

2. Eliminate simpler solutions

Text and static graphics are:

  • faster and easier to produce than custom code
  • more universally accessible across platforms and devices

Therefore, interactives must be reserved for stories that cannot be told any other way.

Signaling the percieved importance of a story is not a justification for interactivity (or large static graphics!)

3. Use data integral to the story

The data being displyed was analysed to find the story.

It must not consist primarily of data selected after the fact to illustrate a conclusion reached by other reporting techniques.

4. Build something of lasting value

Introduce functionality or tell an ongoing story that can attract readers for longer than any one news item, ideally indefinitely.

Rule of thumb: It should retain value to readers for longer than it takes to produce.

5. Involve interactive from the start

Reporters, not editors, must initiate work on interactive projects.

Once copy has been filed, it is too late to develop an interactive.

Our job is to make stories better, not prettier.

Characteristics of great interactives

Great interactives also have at least one of these:

  • Provides a narrative with both a ‘macro’ and a ‘micro’ view
  • Automates an repetitive editorial process
  • Integrates social media
  • Uses data outputs beyond graphics
  • Furthers wider stategic objectives of FT.com
    (eg responsive design or navigation improvements)
